# ADR — Architecture Decision Records

ここには Foldex の設計判断の履歴を残す。
仕様書 (`FOLDEX-HANDOFF.md`) は「いま何が確定しているか」を書く場所。
ADR は「なぜそう決めたか・他の選択肢は何だったか」を残す場所。

## いつ書くか

- 大きめの設計判断をするとき (依存ライブラリの選定、アーキ方針の変更など)
- 仕様書に書くほどではないが、後から「なんでこうしたんだっけ?」となりそうなとき

## 命名

`docs/ADR/0001-<短いケバブ名>.md` のように4桁ゼロ埋めで連番を振る。

## テンプレート

```markdown
# ADR-0001: <決めたことの一行サマリ>

- ステータス: 提案中 / 採用 / 廃棄 / 置き換え (ADR-XXXX)
- 日付: YYYY-MM-DD

## 背景
何が課題で、なぜ判断が必要なのか。

## 検討した選択肢
- 案A: ...
- 案B: ...
- 案C: ...

## 判断
どれを採ったか、なぜか。

## 影響
- ポジティブ: ...
- ネガティブ / トレードオフ: ...

## 関連
- FOLDEX-HANDOFF.md §...
- 関連 ADR: ...
```
